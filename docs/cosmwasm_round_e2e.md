# CosmWasm AMACI Round E2E Cost Harness

`crates/cosmwasm-amaci-round` is a measurement contract for the full AMACI
round flow. It enforces the same high-level proof consumption order as the
zkSTARK round tests:

```text
processDeactivate -> addNewKey -> processMessages[*] -> tally[*]
```

Each step verifies an SP1 compressed proof with
`SP1CompressedVerifierRaw::verify_with_public_values`, then advances the round
state. The contract is intentionally an E2E cost harness, not the production
AMACI business contract.

Build the contract:

```bash
npm install
npm run build:round-contract
```

Run against a local `dorad` devnet:

```bash
npm run round:e2e:local
```

The script writes:

```text
round-e2e-results/<stamp>/summary.json
round-e2e-results/<stamp>/summary.md
```

`costGasPricePeaka` controls the DORA estimate. The local devnet can sign with
`signGasPricePeaka=0` while still reporting estimated cost at the target chain
gas price.

When high-performance-machine artifacts are available for every proof stage,
replace each `stages[*].msgPath` with the corresponding generated
`verify-compressed.msg.json` file. For the five-signup zkSTARK-style flow, set:

```json
{
  "expected": {
    "process_deactivate": 1,
    "add_new_key": 1,
    "process_messages": 1,
    "tally": 2
  }
}
```

## Real Five-Signup Round

The real round fixture is generated by Rust, not hand-written JSON:

```bash
npm run fixture:five-signup
```

It writes `fixtures/five-signup-round/` with five stage inputs:

```text
processDeactivate
addNewKey
processMessagesFull
tally0
tally1
```

The fixture data is:

- 5 original signups: state indices `0..4`.
- 2 deactivate messages: state indices `3` and `4`.
- 1 addNewKey: old state index `4`, new state index `5`.
- 5 vote messages:
  - state `0`, option `0`, weight `1`, valid.
  - state `0`, option `4`, weight `5`, valid.
  - state `3`, option `1`, weight `1`, invalid because the old key is deactivated.
  - state `4`, option `2`, weight `2`, invalid because the old key is deactivated.
  - state `5`, option `4`, weight `5`, valid with the new key.
- Expected raw result: `[1, 0, 0, 0, 10]`.

The core test `five_signup_round_fixture_executes_and_links_public_state`
executes all five stage inputs and checks the important public-output links:
deactivate root into addNewKey, processMessages final state commitment into
tally, and tally commitment between tally batches.

After the proving machine has generated all five compressed proof messages, run:

```bash
npm run build:round-contract
node scripts/run_cosmwasm_round_e2e.mjs \
  --manifest fixtures/round-e2e.five-signup.example.json
```

`fixtures/round-e2e.local.example.json` still exists as a cheap verifier-cost
smoke test. It may point every stage at one downloaded proof message. Use
`fixtures/round-e2e.five-signup.example.json` for the real round.
